Guía de Contribución: Sube tu Proyecto

¡Hola, Problem Solver! Esta guía detalla las especificaciones técnicas necesarias para incorporar la landing page de tu proyecto al escaparate central de CPS Proyectos (cpsproyectos.com).

Seguir estas reglas garantiza que la plataforma se mantenga rápida, premium, segura y fácil de mantener para toda la comunidad.

1. Estructura de Carpetas

Cada proyecto debe alojarse en su propia subcarpeta dentro de la raíz del repositorio. El nombre de esta carpeta definirá la URL de tu landing page:

cpsproyectos/             <-- Raíz del repositorio central
│
├── agro/                 <-- Subcarpeta del proyecto "Agricultura Resiliente IA"
│   └── index.html        <-- Punto de entrada principal
│
├── tu-proyecto/          <-- [NUEVO] Carpeta del proyecto (minúsculas y guiones)
│   ├── index.html        <-- [NUEVO] Punto de entrada principal (obligatorio)
│   └── assets/           <-- [OPCIONAL] CSS, JS o imágenes locales
│       ├── style.css
│       └── logo.png
│
├── index.html            <-- Home central (debes añadir tu tarjeta aquí)
└── style.css             <-- Estilos de la Home central

2. Estándares Técnicos de la Landing Page

Para que tu proyecto sea aprobado e integrado, la landing page (index.html) de tu subcarpeta debe cumplir con los siguientes requisitos:

A. Autocontenido y Rutas Relativas

  • Activos Locales: Si creas archivos CSS, JS o imágenes locales dentro de tu subcarpeta, debes enlazarlos utilizando rutas relativas (por ejemplo: href="assets/style.css" o src="logo.png").
  • NO usar rutas absolutas locales: Evita enlazar activos usando barras diagonales iniciales (como /assets/style.css) ya que esto romperá los enlaces cuando tu landing page se despliegue en un subdirectorio (como cpsproyectos.com/tu-proyecto/).
  • Empaquetado recomendado: Si tu landing page es sencilla, te recomendamos incluir los estilos en un bloque <style> y los scripts en un bloque <script> dentro del mismo HTML. Esto optimiza la velocidad de carga.

B. Recursos Externos (CDNs y APIs)

  • Se fomenta el uso de librerías o tipografías externas mediante CDNs (como Google Fonts, FontAwesome, Chart.js, etc.).
  • Al usar enlaces externos, especifica siempre el protocolo seguro https:// de forma explícita. Evita URLs sin protocolo (como //cdnjs.cloudflare.com/...) para prevenir problemas de contenido mixto en el navegador.

C. Diseño Premium y Responsivo

  • La interfaz debe ser completamente adaptativa (responsiva) y verse impecable en móviles, tablets y ordenadores portátiles o de escritorio.
  • Se recomienda mantener una identidad visual sofisticada y limpia. Utiliza fuentes modernas de Google Fonts (como Inter, Outfit o Playfair Display). Evita colores planos chillones y tipos predeterminados del navegador.
  • Sin placeholders rotos: No utilices imágenes genéricas rotas o de prueba. Usa recursos de alta calidad alojados en servicios de stock libres de derechos (como Unsplash).

D. SEO y Metadatos Básicos

Tu archivo index.html debe incluir en la cabecera (<head>) las siguientes etiquetas descriptivas:

<title>Nombre de tu Proyecto — Eslogan descriptivo</title>
<meta name="description" content="Descripción detallada de 1 o 2 líneas sobre la solución técnica que ofrece tu proyecto.">
Requisito de Seguridad

Asegúrate de no subir credenciales, tokens de API o contraseñas de bases de datos al repositorio en texto plano. Utiliza variables de entorno o simula las respuestas en el frontend si es una demo estática.

3. Proceso para Añadir tu Tarjeta al Showcase

Una vez que tengas la landing page de tu proyecto lista, debes dar de alta tu tarjeta en la Home del Showcase (index.html en la raíz):

  1. Abre el archivo index.html de la raíz del proyecto.
  2. Localiza la sección <div class="projects-grid">.
  3. Duplica y añade un bloque <article class="project-card"> como este:
<article class="project-card">
  <div class="card-header-img" style="background-image: url('https://images.unsplash.com/...')">
    <span class="card-badge badge-dev">Activo</span>
  </div>
  <div class="card-content">
    <h3 class="card-title">Nombre de tu Proyecto</h3>
    <p class="card-description">
      Breve resumen (máximo 3 líneas) del problema real que resuelve la herramienta.
    </p>
    <div class="card-tech-stack">
      <span class="tech-tag">Python</span>
      <span class="tech-tag">Geolocalización</span>
    </div>
    <div class="card-footer">
      <a href="tu-carpeta/" class="card-btn card-btn-primary">
        Ver Demo <i class="fas fa-arrow-right"></i>
      </a>
      <a href="https://github.com/domoprac/tu-proyecto" target="_blank" class="card-btn card-btn-secondary">
        <i class="fab fa-github"></i> Repositorio
      </a>
    </div>
  </div>
</article>

4. Flujo de Publicación (GitHub)

  1. Haz un Fork del repositorio de la comunidad: github.com/domoprac/cpsproyectos.
  2. Clona tu fork localmente y crea una rama de trabajo: git checkout -b feature/nombre-proyecto.
  3. Añade tu carpeta de proyecto y realiza la modificación en el index.html de la raíz.
  4. Confirma tus cambios (git commit -am "feat: añade proyecto nombre-proyecto") y súbelos a tu fork.
  5. Abre un Pull Request (PR) desde GitHub hacia la rama main del repositorio original de domoprac/cpsproyectos.
  6. Una vez revisado y aprobado por los administradores de la comunidad, el despliegue automático publicará tu landing en cuestión de minutos.
